Seminar Overview & Mechanics

Seminar Objective

The Core Objective: This seminar studies the interface between traditional time-series econometrics and modern machine learning (ML) for (i) forecasting and (ii) policy-relevant inference.

Central Themes:

  • Treating ML methods as explicit optimization/statistical objects, not black boxes.
  • Proper evaluation under strict time-series constraints: serial dependence, nonstationarity, structural breaks, and data revisions.
  • Shifting safely from structural identification to algorithmic risk minimization.

Course Mechanics & Evaluation

Format

  • 6 Lectures, 3 hours each.
  • Structure per Lecture: Theory & derivations (~75 min), Paper discussion (~75 min), and a Coding lab component (~30 min).
  • Tools: Python or R.

Evaluation Metrics

Component Weight Description
Memos & Participation 25% 1-page reading memo due before each Lecture.
Presentation 25% Present one syllabus paper + a 10-minute code walkthrough.
Final Project 50% 10–15 page replication paper or research proposal. Includes a complete empirical pipeline and honest OOS evaluation.

The Seminar Project

A Fixed Evaluation Protocol: To compare methods fairly without getting lost in “cool models”, the entire 6-week seminar runs on a single canonical forecasting task. Every new model answers the same question under the exact same rules.

The Task: Multi-horizon forecasting of YoY Core CPI Inflation.

  • Baselines: Mean/AR, small VAR, (optionally BVAR).
  • Data-Rich Extension: Later lectures will inject FRED-style predictor panels.
  • Evaluation Rule: Strict rolling/expanding OOS, leakage-safe tuning.
  • Horizons: \(h = \{1, 3, 6, 12\}\).
  • Metrics / Tests:
    • Forecast loss: RMSE / MAE by horizon.
    • Comparison: Diebold-Mariano test when appropriate.
    • Classification tasks: AUC / PR + calibration.
